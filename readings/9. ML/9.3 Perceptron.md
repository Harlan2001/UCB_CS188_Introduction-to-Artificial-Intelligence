# 9.3 感知机（Perceptron）

## 9.3.1 线性分类器（Linear Classifiers）

朴素贝叶斯的核心思想是从训练数据中提取某些属性，称为特征（features），然后估计给定特征的标签概率：

\[
P(y \mid f_1, f_2, \dots, f_n)
\]

因此，给定一个新的数据点，我们可以提取相应的特征，并将该数据点分类为特征概率最高的标签。  
然而，这需要我们估计概率分布，我们在之前使用最大似然估计（MLE）做了这个。  

如果我们不去估计概率分布，而是直接做分类呢？  
让我们从一个简单的线性分类器开始，它可用于二分类问题，其中标签只有两种可能：正类（positive）或负类（negative）。

---

线性分类器的基本思想是使用特征的线性组合进行分类——这个值称为 **激活值（activation）**。  
具体而言，激活函数接收一个数据点，将数据点的每个特征 \(f_i(x)\) 与对应的权重 \(w_i\) 相乘，并输出所有结果的和。  
用向量形式表示，这可以写成权重向量 \(w\) 与特征向量 \(f(x)\) 的点积：

\[
\text{activation}_w(x) = h_w(x) = \sum_i w_i f_i(x) = w^T f(x) = w \cdot f(x)
\]

---

如何用激活值进行分类？  
对于二分类问题，当数据点的激活值为正时，我们将其分类为正类；当激活值为负时，分类为负类：

\[
\text{classify}(x) =
\begin{cases} 
+, & \text{if } h_w(x) > 0 \\
-, & \text{if } h_w(x) < 0
\end{cases}
\]

---

从几何角度理解，让我们重新看向向量化的激活函数。  
点积可以写为：

\[
h_w(x) = w \cdot f(x) = \|w\| \|f(x)\| \cos(\theta)
\]

![](../../image/9.%20ML/linear_classifier_fig3.png)

其中 \(\|\cdot\|\) 表示向量的模（大小），\(\theta\) 是 \(w\) 与 \(f(x)\) 之间的夹角。

因为模长总是非负的，而分类规则看的是激活值的符号，所以决定类别的关键仅是 \(\cos(\theta)\)：

\[
\text{classify}(x) =
\begin{cases} 
+, & \text{if } \cos(\theta) > 0 \\
-, & \text{if } \cos(\theta) < 0
\end{cases}
\]

---

我们关心的是 \(\cos(\theta)\) 为正还是负。  
显然当 \(\theta < \pi/2\) 时，\(\cos(\theta) \in (0,1]\)，为正；  
当 \(\theta > \pi/2\) 时，\(\cos(\theta) \in [-1,0)\)，为负。  

直观上，这意味着我们的简单线性分类器是在检查新数据点的特征向量是否大致“指向”预定义的权重向量方向：  
- 如果方向相同，则分类为正类。  
- 如果方向相反，则分类为负类。

\[
\text{classify}(x) =
\begin{cases} 
+, & \text{if } \theta < \pi/2 \ (\text{锐角}) \\
-, & \text{if } \theta > \pi/2 \ (\text{钝角})
\end{cases}
\]

---

到目前为止，我们还没有考虑激活值为零的情况：

\[
\text{activation}_w(x) = w^T f(x) = 0
\]

对于这些点，有 \(\cos(\theta) = 0\)，即 \(\theta = \pi/2\)（90°）。  
换句话说，这些数据点的特征向量与权重向量 \(w\) 正交。  
我们可以画一条虚线，正交于 \(w\)，任何在该线上特征向量的激活值为 0：

**决策边界（Decision Boundary）**  

![](../../image/9.%20ML/linear_classifier_fig4.png)

这条蓝线称为 **决策边界**，它将分类为正类的区域与分类为负类的区域分开。  
在高维空间中，线性决策边界通常称为 **超平面（hyperplane）**，它是比潜在空间低一维的线性表面，将空间划分为两部分。  
对于一般分类器（非线性），决策边界可能不是线性的，但仍定义为特征向量空间中分隔不同类别的表面。  
对于位于决策边界上的点，可以任意应用一个标签（在以下算法中，我们将其归为正类）。
